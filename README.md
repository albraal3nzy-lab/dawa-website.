# موقع دواء (Dawa)

موقع لعرض توفر الأدوية بالصيدليات — نسخة ويب مبنية بـ React + Vite.

## التشغيل محلياً

```bash
npm install
npm run dev
```

بيفتح الموقع على `http://localhost:5173`

## النشر على Vercel

1. ارفع هاد المجلد على GitHub (مستودع جديد)
2. روح على vercel.com → New Project → اختار المستودع
3. Vercel رح يتعرف تلقائياً إنه مشروع Vite، بس تأكد:
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
4. اضغط Deploy — وخلص، رح ياخدلك رابط مباشرة (مثلاً `dawa.vercel.app`)

## النشر على Netlify

1. ارفع المجلد على GitHub
2. روح على netlify.com → Add new site → Import from Git
3. حدد:
   - **Build Command:** `npm run build`
   - **Publish Directory:** `dist`
4. Deploy

## ملاحظات مهمة قبل ما تنشره فعلياً

هاد حالياً **نموذج تجريبي (demo)** — البيانات (الصيدليات، المرضى، الوصفات) محفوظة بالذاكرة بس (state)، يعني:
- أي تحديث (إضافة دواء، فرع، حساب جديد) بيروح لما تسكر الصفحة أو تعمل reload
- ما في تسجيل حقيقي أو حماية لكلمات المرور
- رمز التحقق (OTP) معروض على الشاشة مباشرة للتجربة، مش مُرسل فعلياً برسالة SMS

**لو بدك موقع فعلي يشتغل بشكل حقيقي**، لازم تضيف:
- قاعدة بيانات حقيقية (مثل Supabase أو Firebase) لحفظ الصيدليات والمرضى والمخزون
- نظام إرسال SMS حقيقي لرمز التحقق (مثل Twilio)
- تشفير حقيقي لكلمات المرور وباك اند يتحقق من تسجيل الدخول
- دومين خاص (بدل ما يبقى رابط Vercel/Netlify الافتراضي)

قوللي إذا بدك أساعدك بأي وحدة من هاي الخطوات.
